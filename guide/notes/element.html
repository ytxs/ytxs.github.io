    # element组件日常爬坑

    # 1. form表单的label的位置

    举个栗子：

    有些label是在输入框或者下拉框上面的

    而有些label则是在下拉框或者输入框左侧的

    根据label-position 这个属性去控制 值可为 top、left、right，当设为 top 时标签会置于表单域的顶部，left/right则表示label的对齐方式是左对齐还是右对齐

    <el-form :label-position="labelPosition" label-width="80px" :model="formLabelAlign">
  <el-form-item label="名称">
    <el-input v-model="formLabelAlign.name"></el-input>
  </el-form-item>
  <el-form-item label="活动区域">
    <el-input v-model="formLabelAlign.region"></el-input>
  </el-form-item>
  <el-form-item label="活动形式">
    <el-input v-model="formLabelAlign.type"></el-input>
  </el-form-item>
</el-form>

    # 2. 关于分页的一个坑

    具体描述：

    在中间页码选择一个筛选条件，然后进行查询。此时是将页码重置为1了的，也就是说有了筛选条件之后的查询函数中的页码是为1的而不是当前页的页码。当数据返回之后，再次跳转到刚才的页码，有时候会跳不过去甚至有时候下一页点击之后无法触发页码变化的函数。

    解决方法：

    具体请参考文档：https://github.com/ElemeFE/element/issues/11007

    获取分页组件的实例 将lastEmittedPage这个参数也设置为1就ok了

    eg：

    this.$refs.pagination.lastEmittedPage = 1
    1

    若不重置为1， 并且后端没有进行特殊处理的话，后端会按照前端传过去的页码进行查询。可能会发生的情况是：

    根据该筛选条件，查出来是有数据的，但是数据量可能达不到你当前的页码，所以就会出现total有数据，但是页面是空白的。
